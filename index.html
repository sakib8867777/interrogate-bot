Flow
How It Works

Integrate payments in five steps. The middleware handles all complex gateway routing for you.

1

Initiate Request

Your bot sends a JSON payload to POST /api with user details, amount, webhook URL, and your secret key.

2

Gateway Creation

The Middleware authenticates and creates the payment intent on the SbPAY Gateway. A unique payment_url is returned immediately.

3

User Checkout

Send payment_url to the user as a Telegram inline keyboard button. They tap it and complete the transaction on SbPAY checkout.

4

Verification

The Gateway processes the payment and returns the final status to the Middleware for server-side cryptographic verification.

5

Webhook Delivery

The Middleware fires a POST webhook to your bot server with {"ok": true} or {"ok": false}. Deliver goods instantly.

Setup
Getting Your API Key

The key field in every API request is your SbPAY Brand Key. Follow these steps to obtain it.

1

Register at SbPAY

Visit teamsbapp.com and create a free account. Registration takes under a minute.

2

Activate the Free Plan

After logging in, go to Plans and activate the 12-month free plan. This is required to unlock gateway features.

3

Create a Wallet

Navigate to the Wallet section and set up your wallet. This is where funds from payments will be collected.

4

Add a Brand

Go to Brands and create a new brand for your project. Give it a name that represents your bot or service.

5

Add a Device / Domain

Under your brand, add a Device or domain. This registers your middleware endpoint with the gateway.

6

Copy Your Brand Key

Inside your Brand settings, you will see a Brand Key. This is your key / secret API key. Copy it and store it securely as an environment variable in your bot.

 Keep it secret

Your Brand Key is a sensitive credential. Never paste it directly into source code, commit it to Git, or share it publicly. Always load it from environment variables.

API Reference
Endpoint Documentation

Complete reference for integrating the SbPAY Middleware into your application.

POST/api

Create a new payment intent and generate a secure checkout URL. The single endpoint your bot calls to initiate any transaction.

Request Body Parameters

webhookstringrequired
Absolute HTTPS URL where the payment result will be POSTed to your bot.
namestringrequired
Customer display name shown on the checkout page.
keystringrequired
Your secret API brand key. Store in environment variables only — never hardcode.
amountstringrequired
Transaction amount in default currency. Pass as string, e.g. "500".
user_idstringrequired
Telegram user ID as string — links payment history to the user account.
domainstringrequired
Your SbPAY gateway domain. Example: "https://pay.teamsbapp.com"
JSON — Request Body
{
  "webhook": "https://your-bot.com/webhook",
  "name": "Customer Name",
  "key": "••••••••••••••••••••••",
  "amount": "500",
  "user_id": "123456789",
  "domain": "https://pay.teamsbapp.com"
}
JSON — Response 200 OK
{
  "ok": true,
  "id": "9384758293",
  "payment_url": "https://gateway-bot-intoget.vercel.app/pay/9384758293",
  "status": "creating"
}
GET/pay/{id}

User-facing checkout endpoint. Send the user to the payment_url returned from POST /api. Redirects automatically to SbPAY checkout.

Payment Status Values

completed failed cancelled verify_failed webhook_failed creating ready
 Tip

Send payment_url as a Telegram inline keyboard button. The user taps it inside Telegram's browser and is redirected automatically to the SbPAY checkout.

Telegram Bot Flow
1. Receive payment_url from POST /api
2. Send Inline Keyboard button to user:
   [ Pay Now ] → URL: payment_url
3. User taps → redirected to SbPAY checkout
4. Payment completes → webhook fired to your bot
WEBHOOKCallback Payload

Once the transaction resolves, the middleware fires a POST request to your specified webhook URL. Listen here to fulfill orders or update balances.

Payload Fields

okbooleantrue
Payment completed and verified successfully.
okbooleanfalse
Payment failed, cancelled, or could not be verified.
 Security
  • Always use HTTPS endpoints for your webhook listener.
  • Never expose your secret API key in client-side code.
  • Verify user details internally before fulfilling orders.
  • Store processed payment IDs to prevent duplicate fulfillment.
Success Payload
{ "ok": true }
Failed / Cancelled Payload
{ "ok": false }

Integration Examples

Quick-start templates for popular platforms. Copy, paste, customize.

Node.js (node-telegram-bot-api)

Uses node-telegram-bot-api and native fetch() to create a payment and send it as an inline keyboard button.

 Note

user_id should be msg.chat.id.toString() — this powers the payment history on checkout.

JavaScript (Node.js)
const TelegramBot = require('node-telegram-bot-api');
const bot = new TelegramBot(process.env.TG_TOKEN, { polling: true });

bot.onText(/\/pay/, async (msg) => {
  const chatId = msg.chat.id;

  const response = await fetch('https://gateway-bot-intoget.vercel.app/api', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      webhook: "https://your-bot.com/webhook",
      name:    msg.chat.first_name || "User",
      key:     process.env.SbPAY_SECRET_KEY,
      amount:  "500",
      user_id: chatId.toString(),
      domain:  "https://pay.teamsbapp.com"
    })
  });

  const data = await response.json();

  if (data.ok) {
    bot.sendMessage(chatId, "Click below to complete your secure payment:", {
      reply_markup: {
        inline_keyboard: [[
          { text: "Pay Now — 500 BDT", url: data.payment_url }
        ]]
      }
    });
  } else {
    bot.sendMessage(chatId, "Could not create payment. Please try again.");
  }
});

Python (python-telegram-bot v20+)

Uses python-telegram-bot v20+ async library with httpx for the API call. Fully async, production-ready.

 Best Practice

Load your key from os.environ["SbPAY_SECRET_KEY"] — never hardcode secrets in source files.

Python (async)
import os
import httpx
from telegram import Update, InlineKeyboardButton, InlineKeyboardMarkup
from telegram.ext import ApplicationBuilder, CommandHandler, ContextTypes

MIDDLEWARE_URL = "https://gateway-bot-intoget.vercel.app/api"

async def pay(update: Update, context: ContextTypes.DEFAULT_TYPE):
    user = update.effective_user
    payload = {
        "webhook": "https://your-bot.com/webhook",
        "name":    user.first_name or "User",
        "key":     os.environ["SbPAY_SECRET_KEY"],
        "amount":  "500",
        "user_id": str(user.id),
        "domain":  "https://pay.teamsbapp.com"
    }
    async with httpx.AsyncClient() as client:
        res  = await client.post(MIDDLEWARE_URL, json=payload, timeout=15)
        data = res.json()

    if data.get("ok"):
        keyboard = [[InlineKeyboardButton("Pay Now", url=data["payment_url"])]]
        await update.message.reply_text(
            "Click below to complete your payment:",
            reply_markup=InlineKeyboardMarkup(keyboard)
        )
    else:
        await update.message.reply_text("Payment creation failed. Try again.")

app = ApplicationBuilder().token(os.environ["TG_TOKEN"]).build()
app.add_handler(CommandHandler("pay", pay))
app.run_polling()

PHP (cURL)

Standard PHP cURL implementation. Ideal for traditional backend architectures, WordPress, or Laravel projects.

 Best Practice

Store your key in .env and load with vlucas/phpdotenv. Never commit secrets to version control.

PHP (cURL)
<?php
$payload = json_encode([
    "webhook" => "https://your-bot.com/webhook",
    "name"    => "Customer Name",
    "key"     => $_ENV['SbPAY_SECRET_KEY'],
    "amount"  => "500",
    "user_id" => "123456789",
    "domain"  => "https://pay.teamsbapp.com"
]);

$ch = curl_init('https://gateway-bot-intoget.vercel.app/api');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, $payload);
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Content-Type: application/json']);

$response = curl_exec($ch);
curl_close($ch);

$result = json_decode($response, true);
if ($result['ok']) {
    echo "Payment URL: " . $result['payment_url'];
}
?>

BJS — bots.business Platform

Full deposit flow for the bots.business platform using custom BJS (Bot JavaScript). Five commands handle the complete payment lifecycle.

 Required Library

Before using this integration, open your bot in bots.business, go to the Library tab at the bottom, and install the Webhooks library. It is required for Libs.Webhooks.getUrlFor() to work.

 About BJS

BJS is a custom JavaScript dialect used in the bots.business platform. Commands are triggered by Telegram messages. Wait for answer: true means the command waits for the next user input before proceeding.
